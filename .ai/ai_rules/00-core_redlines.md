# 🛑 强制执行红线 (Core Redlines)

为了确保本模板项目的资产完整性、不发生语义漂移和意外故障，任何 AI 助手在辅助开发时，务必牢记并严格遵守以下跨 IDE 的强制红线：

## 1. 模板资产保护红线 (IDE 无关)
- **统一模板（SoT）优先**：`.ai/` 目录属于跨 IDE 的独立中立模板资产，包含了所有规范的 Single Source of Truth (SoT)。
- **严禁删除或同化源文件**：绝对禁止以“格式过时”、“文件迁移”或“IDE 规则升级”等理由，将 `.ai/` 目录下的通用 Markdown/YAML 规则文件删除，并替换成特定 IDE 的专有格式（如 `.mdc`）。
- **禁止破坏性适配**：在将本项目适配到特定的 IDE (如配置 Cursor Rules / Windsurf Rules 等) 时，特定 IDE 的配置文件**只能采用“外壳引用”或“链接（包含说明）”的方式**去指向 `.ai/ai_rules/` 下的真实文件。所有的实质性规则更新必须写回本目录下。

## 2. 代码与工作流行为红线
- **绝对禁止随意篡改 SoT**：除非通过明确的反哺（反馈）流程，否则绝对不能直接修改源文档。
- **永远保持最小化变更**：只修改必要的代码，坚决避免无关代码的重构，修改已有代码必须提供精确的 Diff Patch。
- **验证驱动落地**：生成的任何代码变更必须是可测试和可验证的（对应 DoD）。
- **保护构建链路**：绝不擅自修改任何环境和项目的核心构建配置文件。
- **统一语言**：始终使用中文与用户沟通。
